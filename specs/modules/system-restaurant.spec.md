# Module Spec: Restaurant System

## Purpose
Manages restaurant creation, dish generation, the serving timer, restaurant cleared detection, cycle/restaurant progression, and difficulty scaling modifiers.

## File Path
`src/game/systems/restaurant.ts`

## Dependencies
- `src/types/index.ts` — `GameState`, `Restaurant`, `RestaurantType`, `MeatPart`, `MeatRank`
- `src/game/data/restaurants.ts` — restaurant type definitions, dish counts, serving intervals, rank distribution tables, `RESTAURANT_CYCLE_ORDER`
- `src/game/data/meats.ts` — meat part definitions, vegetable definitions
- `src/game/data/constants.ts` — `SERVING_INTERVALS`, `DISHES_PER_RESTAURANT`, `SERVING_SPEED_REDUCTION_PER_CYCLE`, `SERVING_SPEED_MAX_REDUCTION`, `SWEET_SPOT_REDUCTION_PER_CYCLE`, `SWEET_SPOT_SCALING_CAP_CYCLE`, `PENALTY_INCREASE_PER_CYCLE`, `PENALTY_SCALING_CAP_CYCLE`, `VEGETABLE_INSERT_CHANCE`, `INITIAL_TABLE_CAPACITY`, `INITIAL_GRILL_SLOTS`, `BASE_RESTAURANT_CLEAR_COINS`

## Functions

### `createRestaurant(cycle: number, indexInCycle: number, currentElapsedTime: number): Restaurant`

- **Purpose**: Constructs a fully initialized `Restaurant` object for the given position in the cycle. Applies difficulty scaling to the serving interval. Pre-generates the full serving queue.
- **Preconditions**:
  - `cycle >= 1`
  - `indexInCycle` in `[0, 3]`
  - `currentElapsedTime >= 0` (pass `state.elapsedTime` from the calling engine)
- **Postconditions**:
  - Returns a new `Restaurant` object; pure.
  - `definition.type = RESTAURANT_CYCLE_ORDER[indexInCycle]` (from `data/restaurants.ts`)
  - `totalMeatDishes = DISHES_PER_RESTAURANT[type]`
  - `meatDishesEaten = 0`
  - `dishesServed = 0`
  - `timeSinceLastServe = 0`
  - `startTime = currentElapsedTime` (game-time seconds; used for DPM calculation in `awardQuickTurnoverCoins`)
  - `isCleared = false`
  - `effectiveServingInterval` is calculated by applying difficulty scaling:
    - `reduction = min((cycle - 1) * SERVING_SPEED_REDUCTION_PER_CYCLE, SERVING_SPEED_MAX_REDUCTION)`
    - `effectiveServingInterval = SERVING_INTERVALS[type] - reduction`
    - Floor: `effectiveServingInterval` may not be less than `1.0` (practical minimum). [TUNE]
  - `rankDistribution` = the distribution table for `type` (see table above).
  - Serving queue is generated by calling `generateDish` `totalMeatDishes` times and interleaving vegetables.
    - For each meat slot, independently roll `VEGETABLE_INSERT_CHANCE` to insert a vegetable before the meat dish. The number of vegetables inserted is variable; `totalMeatDishes` tracks meat dishes only.
    - Concrete algorithm: build a list of `totalMeatDishes` meat parts (each via `generateDish`), then for each item in the list roll a Bernoulli variable with probability `VEGETABLE_INSERT_CHANCE`; if true, prepend a random vegetable to the queue at that position.
- **Edge cases**:
  - Cycle 1 has no serving speed reduction (`cycle - 1 = 0`; reduction = 0).
  - `SERVING_SPEED_MAX_REDUCTION = 1.0s` — the cap is reached at cycle 3 (`min((3-1) * 0.5, 1.0) = 1.0`). Formula: `min((cycle - 1) * SERVING_SPEED_REDUCTION_PER_CYCLE, SERVING_SPEED_MAX_REDUCTION)` where `SERVING_SPEED_MAX_REDUCTION = 1.0` and `SERVING_SPEED_REDUCTION_PER_CYCLE = 0.5`. [TUNE]

### `generateDish(restaurant: Restaurant): MeatPart`

- **Purpose**: Picks one random meat part from the parts available at the restaurant's rank distribution. Vegetables are NOT returned by this function — they are inserted separately by the queue builder in `createRestaurant`.
- **Preconditions**:
  - `restaurant.definition.rankDistribution` weights sum to 1.0.
- **Postconditions**:
  - Returns a `MeatPart`; pure.
  - Roll rank using the distribution: pick a random number in `[0, 1)` and find the bracket it falls into.
  - From all meat parts of the selected rank, pick one uniformly at random.
  - Return the selected `MeatPart`.
- **Edge cases**:
  - Rank with weight `0` is never selected.
  - Elite meat only appears in Boss restaurants (weight `0` everywhere else).
  - Common meat never appears in High-End restaurants (weight `0`).
  - If a rank has no parts defined (shouldn't happen with valid data), throw a data-integrity error.

### `tickServing(state: GameState, deltaTime: number): GameState`

- **Purpose**: Advances the serving timer. When the interval elapses, dequeues the next dish from the serving queue and calls `serveDish` (in the grilling system). Handles the case of multiple dishes becoming due in a single tick.
- **Preconditions**:
  - `state.restaurant !== null`
  - `deltaTime > 0`
- **Postconditions**:
  - Returns new `GameState`; no mutation.
  - `state.restaurant.timeSinceLastServe += deltaTime`.
  - While `state.restaurant.timeSinceLastServe >= state.restaurant.effectiveServingInterval` and the serving queue is non-empty:
    - `timeSinceLastServe -= effectiveServingInterval` (consume one interval per dish)
    - Dequeue the next dish from the serving queue
    - Call `serveDish(state)` (from `system-grilling.ts`) to place it on grill or table
    - If `serveDish` triggers a game-over (table overflow), stop processing and return the game-over state immediately.
  - If the serving queue is empty and `timeSinceLastServe >= effectiveServingInterval`: do not loop (no more dishes to serve); `timeSinceLastServe` may retain the remainder.
- **Edge cases**:
  - A very large `deltaTime` (e.g., tab focus returning after a long pause) could cause many dishes to be served at once. The while-loop handles this correctly.
  - If game-over is triggered mid-loop, no further dishes are served.
  - `'quick-order'` skill reduces `effectiveServingInterval` by `1s` [TUNE] for the current restaurant's remaining dishes. This is applied at skill acquisition time by updating `state.restaurant.effectiveServingInterval` directly (handled by the skill system, not here).

### `checkRestaurantCleared(state: GameState): boolean`

- **Purpose**: Returns `true` when all required meat dishes have been eaten for the current restaurant.
- **Preconditions**:
  - `state.restaurant !== null`
- **Postconditions**:
  - Returns `state.restaurant.meatDishesEaten >= state.restaurant.totalMeatDishes`.
  - Pure — no state mutation.
- **Edge cases**:
  - Discarded dishes do NOT count toward `meatDishesEaten`.
  - Vegetables do NOT count toward `meatDishesEaten` or `totalMeatDishes`.
  - Dishes still on the grill or table do not prevent clearing if `meatDishesEaten >= totalMeatDishes`.

---

### `advanceToNextRestaurant(state: GameState): GameState`

- **Purpose**: Moves to the next restaurant in the cycle. Increments `restaurantIndexInCycle` (0→1→2→3→0), incrementing `cycle` when wrapping from Boss (3) back to Chain (0). Resets restaurant state.
- **Preconditions**:
  - `checkRestaurantCleared(state) === true`
- **Postconditions**:
  - Returns new `GameState`; no mutation.
  - `nextIndex = (state.restaurantIndexInCycle + 1) % 4`
  - `nextCycle = nextIndex === 0 ? state.cycle + 1 : state.cycle`
  - Creates a new `Restaurant` via `createRestaurant(nextCycle, nextIndex, state.elapsedTime)`.
  - `state.score` is incremented by 1 (restaurant cleared = +1 score).
  - `state.coins += BASE_RESTAURANT_CLEAR_COINS` (+10 [TUNE]).
  - `state.highestRestaurantTypeReached = max(state.highestRestaurantTypeReached, nextIndex)`.
  - `'regular-customer'` skill: if held, `state.staffWarningCount = max(0, state.staffWarningCount - 1)` and the warning level is recalculated.
  - `'regular-customer-bonus'` skill: if held and `state.staffWarningCount === 0` at clear time (before any `'regular-customer'` reduction), award +`REGULAR_CUSTOMER_BONUS_COINS` coins.
  - The grill and table are NOT cleared — any in-progress items remain.
  - `'quick-turnover-bonus'` skill: if dishes-per-minute for the cleared restaurant exceeds `QUICK_TURNOVER_DPM_THRESHOLD`, award +`QUICK_TURNOVER_BONUS_COINS` coins (computed by `awardQuickTurnoverCoins` in `system-economy`).
- **Edge cases**:
  - True Ending check: if `nextCycle === 5` and `nextIndex === 0` (i.e., cycle 4 Boss was just cleared), trigger True Ending before advancing. [Out of scope for this module — the game engine checks `cycle === 4 && indexInCycle === 3`.]
  - After advancing, `shouldShowNode` (in `system-node.ts`) is evaluated to determine whether a Node screen appears before the next restaurant.

---

### `getDifficultyModifiers(cycle: number): { sweetSpotReduction: number; penaltyMultiplier: number; servingSpeedReduction: number }`

- **Purpose**: Returns the three difficulty scaling values for a given cycle. All values are capped per `GAME_DESIGN.md §7`. Used by other systems (grilling, penalty) to apply scaled values.
- **Preconditions**:
  - `cycle >= 1`
- **Postconditions**:
  - Pure function; no state mutation.
  - `servingSpeedReduction`:
    - **Canonical formula**: `servingSpeedReduction = min((cycle - 1) * SERVING_SPEED_REDUCTION_PER_CYCLE, SERVING_SPEED_MAX_REDUCTION)`
    - Where `SERVING_SPEED_REDUCTION_PER_CYCLE = 0.5` and `SERVING_SPEED_MAX_REDUCTION = 1.0`. Cap is reached at cycle 3. [TUNE]
  - `sweetSpotReduction`:
    - `sweetSpotReduction = min((cycle - 1) * SWEET_SPOT_REDUCTION_PER_CYCLE, (SWEET_SPOT_SCALING_CAP_CYCLE - 1) * SWEET_SPOT_REDUCTION_PER_CYCLE)`
    - (4 steps from cycle 1→5 = −1.2s maximum)
  - `penaltyMultiplier`:
    - `penaltyMultiplier = 1.0 + min(cycle - 1, PENALTY_SCALING_CAP_CYCLE - 1) * PENALTY_INCREASE_PER_CYCLE`
    - e.g., cycle 1 → `1.0`, cycle 2 → `1.1`, cycle 5 → `1.4`
- **Edge cases**:
  - Cycle 1: all reductions = 0; multiplier = 1.0 (no scaling applied in the first cycle).
  - Values are always non-negative; no underflow risk.

## State Transitions

### Restaurant Cycle Order

```
Cycle N:  Chain (0) → Local (1) → High-End (2) → Boss (3)
                                                       │
                                            advanceToNextRestaurant
                                                       │
                                                       ▼
Cycle N+1: Chain (0) → Local (1) → High-End (2) → Boss (3) → ...
```

### Restaurant Cleared Flow

```
meatDishesEaten reaches totalMeatDishes
  └─► checkRestaurantCleared() = true
        └─► Skill selection screen (3 choices — always)
              └─► shouldShowNode(cycle, restaurantsCompletedInCycle) ?
                    ├─► YES → Node screen (Shop or Rest)
                    └─► NO  → advanceToNextRestaurant() directly
```

## Invariants

1. `restaurantIndexInCycle` is always in `[0, 3]`.
2. `cycle` only increments when `indexInCycle` wraps from 3 to 0.
3. `meatDishesEaten` never exceeds `totalMeatDishes`.
4. The serving queue contains exactly `totalMeatDishes` meat dishes plus a variable number of interleaved vegetables at creation time.
5. `timeSinceLastServe` is always `>= 0`.
6. `generateDish` never returns a vegetable; vegetables are inserted separately by the queue builder.
7. `getDifficultyModifiers` is a pure function — same `cycle` always returns the same result.
8. `effectiveServingInterval` after difficulty scaling is always `>= 1.0` (practical floor to prevent division-by-zero and degenerate behavior).
9. `state.highestRestaurantTypeReached` is updated in `advanceToNextRestaurant` as `max(previous, nextIndex)` — monotonically non-decreasing.
10. All constant names are imported from `src/game/data/constants.ts` — no local redefinitions.
11. Serving speed cap: maximum reduction is `1.0s`, achieved at cycle 3. Formula: `min((cycle - 1) * 0.5, 1.0)`.
